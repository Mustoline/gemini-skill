# Markdown Best Practices & Standards

All strategic documents produced by the agency must follow these formatting standards to ensure readability and professional presentation.

## 1. Structure and Hierarchy
- **Single H1 per document**: Use `#` only for the main title.
- **Logical Nesting**: Use `##`, `###`, `####` in sequence. Do not skip levels (e.g., `#` to `###`).
- **Consistent Headers**: Use Sentence case for headers (e.g., "Client involvement," not "Client Involvement").

## 2. Lists and Tables
- **Standard Bullet Points**: Use `*` for top-level bullets and `    *` for nested bullets.
- **Consistent Spacing**: Always leave a blank line before and after a list or table.
- **Table Formatting**: Ensure pipes (`|`) are aligned and headers are separated by a line of dashes (`---`).

## 3. Syntax and Style
- **Emphasis**: Use **bold** for key terms and *italics* for secondary emphasis or quotes.
- **Links**: Use descriptive link text. Never use "click here."
- **Consistency**: Use the same terminology throughout (e.g., don't switch between "User" and "Customer" in the same document).

## 4. Final Polish
- No trailing spaces.
- No consecutive blank lines (more than one).
- End files with a single newline.
